🧩 VENARY EXTENSIONS

Build powerful, self-contained modules for the Venary gaming platform. No core code changes needed.

PHPBB-style Modules Auto-Discovery Hot-Loadable

🏗️ Architecture Overview

Venary uses a PHPBB-style extension system where each module is a self-contained folder inside the extensions/ directory. On server startup, the Extension Loader scans for manifest.json files, validates them, and automatically mounts routes, serves static files, and initializes data collections.

Key Principle: Extensions are completely decoupled from core. They have their own backend routes, frontend pages, CSS, and data — and can be enabled/disabled without any code changes.

📋 Manifest Reference

The manifest.json is the heart of every extension. It declares everything the loader needs:

manifest.json{
  "id": "my-extension",           // Must match folder name
  "name": "My Extension",         // Display name
  "version": "1.0.0",             // Semantic version
  "description": "What it does",  // Shown in admin panel
  "author": "Your Name",          // Author attribution
  "enabled": true,                // Default enabled state

  "nav": [                        // Sidebar navigation entries
    {
      "label": "My Page",         // Link text
      "icon": "grid",             // Icon name (see icon list below)
      "route": "/my-ext",         // Hash route (becomes #/my-ext)
      "position": 5               // Sort order in nav
    }
  ],

  "routes": {
    "prefix": "/api/ext/my-extension",  // API prefix
    "file": "server/routes.js"          // Path to Express router
  },

  "pages": [
    {
      "route": "/my-ext",               // Route this page handles
      "file": "public/pages/my-page.js", // JS file path
      "global": "MyExtPage"              // Global object name defined by the script
    }
  ],

  "css": [
    "public/css/my-style.css"            // Auto-injected stylesheet
  ],

  "data": [
    "my_ext_items",                      // Data collection names (auto-created in DB)
    "my_ext_settings"
  ]
}

Manifest Fields

Field Required Description
id ✅ Unique identifier, must match directory name
name ✅ Human-readable name
version ✅ Semantic version string
description No Description shown in admin
author No Author name

⚙️ Backend Routes

Your backend routes file should export an Express Router. It's mounted at the prefix defined in your manifest (default: /api/ext/<id>).

server/routes.jsconst express = require('express');
const router = express.Router();
const db = require('../../../server/db');
const { authenticateToken } = require('../../../server/middleware/auth');

// Public endpoint
router.get('/items', function(req, res) {
  var items = db._data.my_ext_items || [];
  res.json(items);
});

// Protected endpoint
router.post('/items', authenticateToken, function(req, res) {
  var item = {
    id: require('uuid').v4(),
    user_id: req.user.id,
    title: req.body.title,
    created_at: new Date().toISOString()
  };
  db._data.my_ext_items.push(item);
  db.scheduleSave();
  res.status(201).json(item);
});

module.exports = router;
Important: Use ../../../server/db paths since your file is at extensions/<id>/server/routes.js (3 levels deep from project root).

Available Core Imports

Module Path Provides
Database ../../../server/db db._data, db.scheduleSave()
Auth Middleware ../../../server/middleware/auth authenticateToken, optionalAuth
UUID uuid UUID generation (pre-installed)

🖥️ Frontend Pages

Your page JS file must define a global object with a render(container, params) method. The global name in your manifest tells the loader which window object to look for.

public/pages/my-page.jsvar MyExtPage = {
  async render(container, params) {
    // params is an array of URL segments after your route
    // e.g. #/my-ext/item/123 → params = ['item', '123']

    container.innerHTML = '<div class="my-ext-page">' +
      '<h1>My Extension</h1>' +
      '<div id="my-ext-content"></div>' +
    '</div>';

    // Use the API client for requests
    try {
      var items = await API.get('/api/ext/my-extension/items');
      var content = document.getElementById('my-ext-content');
      content.innerHTML = items.map(function(item) {
        return '<div class="card">' + App.escapeHtml(item.title) + '</div>';
      }).join('');
    } catch (err) {
      App.showToast(err.message, 'error');
    }
  }
};

Function Description

🗄️ Data Storage

List collection names in your manifest's data array. They are auto-created as empty arrays in the JSON data store on first load. Access them via db._data.your_collection.

Tip: Always call db.scheduleSave() after modifying data to persist changes to disk.

🧪 Testing

  1. Place your extension folder in extensions/
  2. Restart the server — look for ✅ Extension loaded: ... in console
  3. Verify your nav link appears in the sidebar
  4. Test your routes with curl or the browser
  5. Check the Admin Dashboard → Extensions tab for status
