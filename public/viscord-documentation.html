Quick Installation

1

Download the Release JAR

Choose the appropriate JAR file for your Minecraft version and mod loader from the repository Releases folder.

2

Install to Server

Place the downloaded JAR file directly into your server's mods folder.

server/
├── mods/
│   └── viscord-1.21.1-neoforge-2.4.4.jar
└── ...
3

Boot Server

Start your Minecraft server to load the Viscord environment. A configuration schema will be generated automatically in config/viscord/viscord.json.

Installation Complete!

Viscord is now hooked into the server. Proceed to the platform setup below.

Discord Platform Setup

1

Create Application

Visit discord.com/developers/applications and create a new application.

2

Configure Intents CRITICAL

In the Bot tab, scroll to Privileged Gateway Intents and enable:

  • MESSAGE CONTENT INTENT (Required for reading chat bridging)
  • SERVER MEMBERS INTENT (Required for account linking)
3

Configure Viscord

Inject your Bot Token, Channel ID, and Webhook URL into the configuration payload:

{
  "discord": {
    "bot_token": "YOUR_BOT_TOKEN",
    "channel_id": "1234567890", 
    "webhook_url": "https://discord.com/api/webhooks/...",
    "invite_url": "https://discord.gg/yourserver"
  }
}

Fluxer Platform Setup

1

Configure General Settings

Set platform: "fluxer" in your configuration to enable the base routing.

"general": {
  "enabled": true,
  "platform": "fluxer"
}
2

Configure Network Routing

Update your Viscord configuration with your Fluxer dashboard settings.

"fluxer": {
  "webhook_url": "YOUR_FLUXER_WEBHOOK",
  "api_key": "YOUR_FLUXER_API_KEY",
  "port": 8080,
  "path": "/webhook"
}
3

Port Forwarding (CRITICAL)

Ensure that port 8080 (or your chosen port) is fully open and port-forwarded on your server machine so the Fluxer API can reach the Minecraft Server.

4

Point Fluxer Dashboard

In your Fluxer provider settings, set the receiver endpoint to http://your-server-ip:8080/webhook

Thread Pool Architecture

Fluxer receiver and WebHook senders operate asynchronously via a CachedThreadPool, protecting the main server thread at all costs. Never lower thread limits unless strictly required.

In-Depth Configuration Options

Basic Settings

Setting Description Default
enabled Master networking toggle for the entire Viscord framework. true
platform The backend engine to use for chat reception (discord or fluxer). "discord"

Tridirectional Engine Settings

When using both Fluxer and Discord simultaneously:

Setting Description Default
tridirectional.enabled Enable the 3-way synchronization system. false
tridirectional.discord_to_fluxer Bridge incoming Discord messages directly to Fluxer channels. true
tridirectional.fluxer_to_discord Bridge incoming Fluxer messages directly to Discord webhooks. true

Text Syntax Mapping

Native parsing engine intercepts Minecraft's underlying Component tags or legacy & sequences and transpiles them tightly into Discord markdown nodes.

Minecraft TagOutcomeDiscord Transpilation

§o / &o Italic *text*
§m / &m Strike ~~text~~

§k / &k ✨ Obscured ✨text✨
Real-time Complex Nested Rendering

Viscord's internal stack parser actively manages markdown open/close tags during transpile to ensure nested formats (e.g. §c§lBold Red) render perfectly as 🔴 **Bold Red**.

Security & Diagnostics

Network Issues

Discord Bot Offline

Validate token hash. Confirm Message Content Intent is physically toggled ON inside the Discord Developer Portal.

Fluxer Binding Failed

Ensure port 8080 (or your configured port) is completely unbound on the physical host machine, and WAN NAT tunneling is enabled if running headless.

Advanced Debug Pipeline

To view deep internal websocket traces and pipeline packet dumps, enable hardware logging inside configuration array:

"advanced": {
  "debug_logging": true
}
